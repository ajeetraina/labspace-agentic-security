SBOM

Software Bill of Materials — your software ingredient list

A complete, machine-readable list of every component in your software — packages, libraries, versions, licenses, and their relationships.

  • Know what is in every image you ship
  • Respond to new CVEs in seconds, not days
  • Required for SLSA compliance and enterprise contracts
# Generate SBOM at build time $ docker buildx build \ --sbom=true \ --provenance=mode=max \ -t myorg/myapp:v1.0 . # Inspect attached SBOM $ docker buildx imagetools inspect \ myorg/myapp:v1.0 # Query via Docker Scout $ docker scout sbom myorg/myapp:v1.0

VEX

Vulnerability Exploitability eXchange — which CVEs actually matter

A machine-readable statement that tells scanners which vulnerabilities are actually exploitable in your context — and which are just noise.

  • Cut CVE noise — suppress false positives automatically
  • Tell scanners which CVEs are not reachable in your code
  • Open standard (OpenVEX), shipped with every Hardened Image
# Export VEX from an image $ docker scout vex get myorg/myapp:v1.0 \ --output vex.json # Feed VEX into Grype (drops noise) $ grype myorg/myapp:v1.0 --vex vex.json # Feed VEX into Trivy $ trivy image --vex vex.json \ myorg/myapp:v1.0

SLSA

Supply-chain Levels for Software Artifacts — provable provenance

A security framework that produces a signed, tamper-evident record of how, where, and from what sources an artifact was built.

  • Prove an artifact's build provenance wasn't tampered with
  • Trace every image back to its source commit and builder
  • Reach SLSA Build Level 3 with signed provenance
# Build with max provenance (SLSA) $ docker buildx build \ --provenance=mode=max \ -t myorg/myapp:v1.0 . # Inspect the SLSA provenance $ docker buildx imagetools inspect \ myorg/myapp:v1.0 \ --format '{{json .Provenance}}' # Verify attestations via Docker Scout $ docker scout attest list myorg/myapp:v1.0
